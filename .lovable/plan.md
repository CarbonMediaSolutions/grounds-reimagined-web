

## Blog Feature with Admin Management

### Overview
Add a Blogs page where visitors can read blog posts, and admins can create/manage them. Blog summaries will be auto-generated using Lovable AI (Gemini).

### Database Changes

**1. Create `blog_posts` table:**
- `id` (uuid, PK)
- `title` (text, not null)
- `summary` (text) — auto-generated by AI
- `content` (text, not null)
- `image_url` (text)
- `published` (boolean, default false)
- `created_at` (timestamptz)
- `updated_at` (timestamptz)
- `author_id` (uuid, references auth.users)

**2. Create storage bucket** `blog-images` (public) for blog post photos.

**3. RLS policies:**
- Anyone can SELECT published posts
- Authenticated admins can INSERT/UPDATE/DELETE

**4. Admin role system:**
- Create `app_role` enum and `user_roles` table with `has_role()` security definer function
- RLS on blog_posts uses `has_role(auth.uid(), 'admin')`

### Edge Function

**`supabase/functions/generate-summary/index.ts`:**
- Accepts blog content, calls Lovable AI (google/gemini-3-flash-preview) to generate a 2-3 sentence summary
- Returns the summary text

### Frontend Pages & Components

**1. `/blogs` — Public blog listing page (`src/pages/Blogs.tsx`):**
- Grid of blog cards showing image, title, summary, date
- Links to individual blog post

**2. `/blogs/:id` — Individual blog post page (`src/pages/BlogPost.tsx`):**
- Full content display with hero image

**3. `/admin/blogs` — Admin blog management (`src/pages/admin/BlogAdmin.tsx`):**
- Protected route (requires admin login)
- List of all posts (published + drafts)
- Create/Edit form with title, content (textarea), image upload
- "Generate Summary" button that calls the edge function
- Publish/unpublish toggle

**4. `/admin/login` — Admin login page (`src/pages/admin/AdminLogin.tsx`):**
- Simple email/password login form

### Navigation
- Add "Blog" link to Navbar between Products and Reach Out
- Admin pages accessible via `/admin/blogs` (not in public nav)

### Auth Flow
- Standard email/password auth for admins
- Admin must be manually assigned the `admin` role in the user_roles table
- Session-based auth check on admin routes
